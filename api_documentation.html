üìã Overview

The Vehicle Classification API provides a robust, production-ready REST interface for classifying vehicle images and generating professional reports. The system analyzes 100√ó90 greyscale vehicle images and predicts 9 distinct attributes with confidence metrics.

9 Classification Dimensions

Make (100), Type (12), Color (10), Decade (70), Country (70), Condition (regression), Stock/Modified (binary), Functional Utility (8), Model (~150)

Batch Processing

Process multiple images in a single request for improved throughput and efficiency.

Professional Reports

Generate executive summaries, recommendations, and detailed confidence metrics in JSON or HTML format.

High Performance

50-100ms per image processing time with intelligent model caching and lazy loading.

üîå API Endpoints

GET
Health Check
Check service health and model availability
/health
Verify that the service is running and models are loaded. Useful for monitoring and health checks.
Response (200):
{ "status": "healthy", "models_ready": true, "cached_reports": 5 }
GET
Model Metadata
Retrieve information about available models
/api/models/metadata
Get comprehensive information about available classifiers and supported attributes.
Response (200):
{ "cached_models": ["make", "type", "color"], "supported_attributes": [...], "version": "1.0" }
POST
Classify Single Image
Classify a single vehicle image
/api/vehicle/classify
Upload a vehicle image and receive predictions for all 9 attributes with confidence metrics.
Request:
multipart/form-data: { "file": <image_file> }
Response (200):
{ "status": "success", "data": { "predictions": {...}, "overall_confidence": 0.87 } }
POST
Batch Classification
Classify multiple images efficiently
/api/vehicle/classify-batch
Upload multiple vehicle images in a single request for batch processing.
Request:
multipart/form-data: { "files": [<file1>, <file2>, ...] }
Response (200):
{ "summary": { "successful": 3, "failed": 0, "average_processing_time_ms": 75.5 }, "results": [...] }
POST
Generate Report
Create a professional classification report
/api/vehicle/report
Generate a comprehensive vehicle classification report with executive summary, recommendations, and confidence metrics.
Request:
multipart/form-data: { "file": <image>, "format": "json|html", "vehicle_id": "optional" }
Response (200):
{ "vehicle_id": "VEH_20260131_143022", "report": { "summary": "...", "confidence_score": 0.85, ... } }
GET
Retrieve Report
Get a previously generated report
/api/vehicle/report/<vehicle_id>
Retrieve a cached report by vehicle ID. Supports JSON and HTML formats.
Query Parameters:
format=json|html (default: json)
Response (200):
{ "status": "success", "data": {...} }

üìä Response Structure

Prediction Object

Field Type Description
attribute string Classification attribute name (Make, Type, Color, etc.)
value string | number Human-readable predicted value
confidence float (0.0-1.0) Confidence score for the prediction
raw_probabilities object Top-K predictions with probability distributions
confidence_metrics object (optional) Detailed uncertainty and confidence analysis

Standard Response Format

Field Type Description
status string "success" or "error"
timestamp string ISO 8601 timestamp
data object Response data (predictions, report, etc.)
message string (optional) Error message if status is "error"

üîç Usage Examples

cURL

Classify single image:
curl -X POST -F "file=@vehicle.jpg" http://localhost:5000/api/vehicle/classify
Generate HTML report:
curl -X POST -F "file=@vehicle.jpg" -F "format=html" http://localhost:5000/api/vehicle/report > report.html
Health check:
curl http://localhost:5000/health

Python Requests

import requests # Classify image with open('vehicle.jpg', 'rb') as f: files = {'file': f} response = requests.post( 'http://localhost:5000/api/vehicle/classify', files=files ) predictions = response.json() print(predictions) # Batch processing files = [('files', open(f'vehicle_{i}.jpg', 'rb')) for i in range(3)] response = requests.post( 'http://localhost:5000/api/vehicle/classify-batch', files=files ) batch_results = response.json() print(f"Processed: {batch_results['summary']['successful']} images")

‚öôÔ∏è Configuration & Constraints

  • Max File Size: 16MB per image
  • Supported Image Formats: JPEG, PNG, BMP, GIF
  • Expected Image Size: 100√ó90 pixels (automatic resizing)
  • Color Format: Greyscale (automatic conversion)
  • Processing Time: 50-100ms per image (GPU-dependent)
  • Batch Size: No limit (depends on available memory)
  • Report Cache: In-memory (cleared on server restart)
  • Concurrent Requests: Supports multi-threaded processing

üìà Performance Tips

  • Batch Processing: Use classify-batch for 10+ images to improve throughput
  • Image Optimization: Pre-resize images to 100√ó90 to avoid server processing
  • Report Retrieval: Cache vehicle_id for later retrieval without re-processing
  • Connection Pooling: Reuse HTTP connections for multiple requests
